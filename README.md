# 修身炉

修身炉是一个知识成长助手。

目标是和我一起积累资料、消化知识、运用知识，并把这些过程变成可追踪、可复用、可继续升级的个人知识系统。

```text
资料进入 -> 消化整理 -> 知识沉淀 -> 运用反馈 -> 再次升级
```

当前版本 Milestone 1 只具备资料积累功能，知识库和自动维护仍是后续方向。


## 使用

添加配置，比如llm的API。

windows系统双击 `run_main.bat` 打开本地控制台，然后按每天的节奏使用：
主页是日常工作的计划制定，任务管理，每日复盘，和明日计划生成。
小红书发布在控制台下拉菜单。

## 理念
1. **一个入口**：用户只在一个界面交互。后台一个主agent，其风格和工作流是用户严选。再通过主agent调用外部功能。
2. **把流程固定**：这个过程，是汲取经验，也是克服遗忘和惰性。人喜欢做熟悉的事。养成习惯和自律很难，那就让程序来辅助。这是修身的本意。
3. **个性化需求**：vibe coding让定制化服务变得简单。但真正的问题是，我的需求是什么？我的个性和习惯是什么？做这个项目本身，就是在梳理我的个性，拓展我的能力。

## Milestone

### 1. 日常资料记录

**理念**：资料记录，人工一定要越简单越好，这样才不会偷懒不记录。而资料的积累和维护，是后期分析自己和工作流的关键。

当前版本：
1. 当下的工作，用最方便的方式记录下来，然后llm按照我个人风格整理记录。
2. 人工写小红书笔记，自动化post到小红书，并可从整篇草稿生成本地封面图。
3. 可启动一个轻量桌面宠物，使用公开宠物素材，并能和鼠标进行简单互动。

这一部分我一直在打磨UI和pipeline，每天都会蹦出新需求，实现需求都会让控制台更懂我更方便。其过程就是在梳理我的需求，固定以及优化我的习惯。

其他软件比如notion，不符合我的记录习惯。我的软件，只需要输入当下在做的事以及想法，软件会自动管理任务列表，而且用我喜欢的风格。所以我只需要专注于当下的事，有需要再去看任务表格以及llm给的建议。

让llm尽可能辅助整理资料，同时用代码约束。

### 2. 知识库建立

根据日常记录，过去笔记，和网络资料，建立一个方便搜索和维护的知识库。
这一阶段须多学习，比如RAG技术的更新。也要参考别人的产品，比如notebook lm

### 3. 自动维护和升级知识

1. llm时代，产品和技术迭代很快。助手能自动化的接收外部资讯，给出自身架构的更新建议。主agent协调cc/codex去做。
2. 每日都会有新知识加入，要动态维护知识库，比如去重合并和索引更新。
3. agent的使用记录，会反映用户的喜好和关注点，据此沉淀知识，并入知识库。
