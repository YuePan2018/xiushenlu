# 修身炉技术储备

把项目相关、暂时不用但未来可能有价值的技术链接或资料整理成可检索 Markdown。该 skill 只服务技术储备，不改控制台、不调用修身炉 pipeline、不写 `data/memory/`。

## 固定位置

- 项目根目录：`C:\Users\Ua Pan\Desktop\project\xiushenlu`
- 技术储备目录：`docs/吸纳/技术储备/`
- 总索引：`docs/吸纳/技术储备/INDEX.md`
- 该目录是 `docs/` 下的自由读写例外；其他 `docs/` 路径仍按项目规则视为受保护路径。

## 工作流

1. 收集用户提供的链接、粘贴文本和补充说明。链接可以读取网页内容；内容过长时只保留链接、必要摘录和摘要，不复制整篇受版权保护正文。
2. 生成一个可检索标题。标题承担关键词/标签作用，应包含技术名、主题和用途，例如 `FastAPI 后台任务与停止机制`。
3. 生成两段内容：
   - `简介`：只介绍当前资料对应的知识、技术、方案或工具本身，不写修身炉未来如何使用。
   - `Agent 快速索引`：说明未来 agent 在什么任务、问题、模块或场景下应该读取这份文档；这里直接包含适用场景。
4. 写入单篇文档：`YYYY-MM-DD_标题安全短名.md`。如果文件已存在，在文件名后追加 `-2`、`-3` 等序号，禁止覆盖旧文档。
5. 更新 `INDEX.md`。索引行字段固定为：文档标题、简介、Agent 快速索引、文件路径、导入日期。
6. 最后回复用户写入了哪些文件，并简短说明标题和索引如何帮助未来搜索。

## 文档格式

单篇文档必须使用以下结构：

```markdown
# 文档标题

## 简介
介绍当前文件对应的知识、技术、方案或工具本身。

## Agent 快速索引
说明未来 agent 在什么任务、问题、模块或场景下应该读取这份文档。

## 原文与链接
保留来源链接、用户粘贴内容或必要摘录。
```

`INDEX.md` 必须使用以下表头；不存在时先创建：

```markdown
# 技术储备索引

| 文档标题 | 简介 | Agent 快速索引 | 文件路径 | 导入日期 |
| --- | --- | --- | --- | --- |
```

索引中的 `简介` 和 `Agent 快速索引` 应是压缩版，适合未来 agent 先读索引筛选候选文档。单篇文档中可以更完整。

## PowerShell 与中文规则

- 检索文件名、目录或内容时，使用 PowerShell 原生命令，不使用 `rg` / ripgrep。
- 读取中文文本文件时，显式使用 `-Encoding utf8`，例如 `Get-Content -Raw -Encoding utf8`。
- 写入 Markdown 时使用 UTF-8。
- 不把 LLM 输出直接当作 shell 命令执行；文件名清洗、重名避让和写入路径必须由 Codex 自己确定。

## 写入细节

- 文件名安全短名：移除 Windows 非法字符 `< > : " / \ | ? *`，把连续空白折叠为单个空格，必要时截断到适合阅读的长度。
- Markdown 表格内容需要转义竖线 `|`，避免破坏索引表。
- `原文与链接` 中优先保留 URL、用户粘贴内容和少量关键摘录；对网页正文以摘要为主。
- 不自动修改 `README.md`、`TECHNICAL.md`、规划文档或 `data/memory/`。
